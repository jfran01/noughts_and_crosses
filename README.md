# noughts_and_crosses

The Odin Project, Project: Tic Tac Toe

Strengths:

- Used classes to ensure methods and variables remain reusable whilst private and relevant
- Avoided creating overly complex/long methods
- Abstracted where possible to prevent code duplication
- Avoided deeply nested loops and over-reliance on external methods
- Used new (to me) built-in methods such as #each_slice, #values_at, and #uniq
- Predicted and took care of edge cases

Improvements:

- Would create a separate group for Board to prevent repeated use of self.board
- Clear up knowledge of when to use self.[instance_variable] vs when to simply use @
